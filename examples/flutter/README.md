# intl-ai Flutter Example

A minimal Flutter app that uses a local `build_runner` builder to translate locale files at build time.

## Plugin code

The builder lives in `examples/flutter/plugin/`. Copy it into your own project and adapt as needed — it is not published as a package.

## Setup

1. Install the `intl-ai` CLI:

   ```bash
   curl -fsSL https://intl-ai.pages.dev/install.sh | sh
   ```

2. Set your API key:

   ```bash
   export OPENAI_API_KEY=sk-...
   ```

3. Get dependencies and run the builder:

   ```bash
   flutter pub get
   flutter pub run build_runner build --delete-conflicting-outputs
   ```

   This invokes `intl-ai fill` and writes translated strings to `assets/locales/es.json`.

4. Run the app:

   ```bash
   flutter run
   ```

## Project layout

```
examples/flutter/
├── plugin/              # local build_runner builder
│   ├── lib/
│   ├── build.yaml
│   └── pubspec.yaml
├── assets/locales/
│   ├── en.json          # source locale
│   └── es.json          # filled by intl-ai
├── build.yaml           # enables the intl_ai builder
├── intl-ai.config.json  # intl-ai configuration
└── lib/
    └── main.dart        # loads JSON assets at runtime
```

## Notes

- `intl-ai.done` is generated by `build_runner`. Add it to `.gitignore`.
- The example uses simple JSON asset loading. For production apps, consider `easy_localization`, `i18n_extension`, or `flutter_gen`.
